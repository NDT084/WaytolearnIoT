Protocoles de Communication IoT

Du bus interne aux réseaux WAN : la fondation de la connectivité dans les systèmes embarqués.

Objectifs pédagogiques : Comprendre les bus internes (I2C, SPI, UART), maîtriser les protocoles IoT (MQTT, CoAP), choisir le bon protocole selon les contraintes (bande passante, consommation, portée), et implémenter des communications fiables dans vos projets IoT.

1. Les Bus Périphériques (Communication Interne)

La communication interne entre le microcontrôleur et ses périphériques (capteurs, mémoires externes, écrans) est gérée par des bus optimisés pour la vitesse et le faible nombre de broches.

Pourquoi plusieurs bus ?

Chaque bus a ses avantages : I2C pour économiser les broches (2 fils), SPI pour la vitesse (écrans, mémoires), UART pour la simplicité (debug, modules). Sur un ESP32, vous pouvez utiliser les 3 simultanément !

I²C

Bus série synchrone maître/esclave. N'utilise que deux fils (SDA & SCL). Idéal pour les petits capteurs I/O.

Caractéristiques :

  • • Vitesse : 100 kHz (standard) à 3.4 MHz (fast mode)
  • • Distance : ~1 mètre (court)
  • • Exemples : Capteurs (BMP280, MPU6050), EEPROM

Code Arduino : Wire.begin(); Wire.write(data);

[Diagramme I2C]

SPI

Bus série synchrone maître/esclave. Plus rapide que I²C, utilise quatre fils. Préféré pour les mémoires flash et les écrans rapides.

Caractéristiques :

  • • Vitesse : Jusqu'à 50+ MHz (très rapide)
  • • Fils : MOSI, MISO, SCK, CS (4 fils minimum)
  • • Exemples : Écrans OLED, cartes SD, Flash externes

Code Arduino : SPI.begin(); SPI.transfer(data);

[Diagramme SPI]

UART

Bus série asynchrone (sans horloge). Utilisé principalement pour le debugging et la communication avec des modules (comme le GPS ou le GSM).

Caractéristiques :

  • • Vitesse : 9600 à 115200 bauds (configurable)
  • • Fils : TX (transmission), RX (réception), 2 fils
  • • Exemples : Serial Monitor, modules GPS, GSM, Bluetooth

Code Arduino : Serial.begin(9600); Serial.println("Hello");

2. Protocoles Applicatifs pour l'Internet des Objets

Au-delà de la couche physique, les protocoles applicatifs sont essentiels pour le cloud. Ils doivent être légers (faible surcharge d'en-tête) et optimisés pour les connexions intermittentes et la faible bande passante.

MQTT

Protocole de messagerie léger basé sur un modèle Publier/S'abonner (Pub/Sub). Idéal pour les communications asynchrones entre dispositifs et le Cloud.

Cas d'usage réel :

Un capteur de température ESP32 publie ses données sur le topic sensors/temperature. Un serveur cloud s'abonne à ce topic et reçoit automatiquement les données. Plusieurs abonnés peuvent recevoir les mêmes données. 

// Exemple ESP32 + MQTT
client.publish("sensors/temp", "25.3");
client.subscribe("commands/led");
  • • QoS : Gère la qualité de service (0, 1, 2) pour garantir la livraison.
  • • Faible Surcharge : Seulement 2 octets d'en-tête minimal.

CoAP

Protocole de transfert de données basé sur le modèle Requête/Réponse (similaire à HTTP, mais optimisé pour l'embarqué). Utilise UDP pour minimiser la taille des paquets.

Cas d'usage réel :

Un capteur CoAP expose une ressource coap://device/temperature. Un autre appareil peut faire une requête GET pour lire la température, ou PUT pour configurer un seuil. Similaire à REST, mais avec seulement 4-10 bytes d'en-tête au lieu de 50+ pour HTTP. 

// Exemple requête CoAP
GET coap://192.168.1.100/temperature
// Réponse : 25.3°C (paquet minimal)
  • • Transfert : Souvent utilisé pour la gestion des ressources et les interactions directes entre appareils.
  • • 6LoWPAN : Fonctionne bien avec ce protocole pour les réseaux maillés IPv6 basse consommation.

3. Les Réseaux Longue Portée (LPWAN)

Pour les applications IoT nécessitant une longue portée et une très faible consommation (plusieurs années d'autonomie), les réseaux LPWAN sont la norme.

Applications LPWAN au Sénégal

Les réseaux LPWAN sont idéaux pour l'agriculture intelligente, la surveillance environnementale, et la gestion de l'eau. Un capteur LoRaWAN peut transmettre des données sur plusieurs kilomètres avec une batterie qui dure 5-10 ans !

  • • Agriculture : Capteurs d'humidité du sol, stations météo
  • • Ville intelligente : Compteurs d'eau, éclairage public
  • • Environnement : Surveillance de la qualité de l'air, suivi de la faune

LoRaWAN

Très longue portée (plusieurs km) et faible bande passante. Utilise des modulations spread spectrum pour la robustesse.

[Architecture LoRaWAN]

6LoWPAN / Zigbee

Protocoles maillés basés sur IPv6 et courts-courriers (quelques mètres). Essentiels pour la domotique et les réseaux denses.
