IoT & Systèmes embarqués • L1 → L3+

L’IoT et les systèmes embarqués, c’est ce qui fait « vivre » les objets connectés.

L’Internet des Objets (IoT) regroupe tous les dispositifs capables de sentir leur environnement, réfléchir via un microcontrôleur et communiquer avec d’autres machines ou le cloud.

Au cœur de ces objets se trouvent les systèmes embarqués : de petits ordinateurs spécialisés (STM32, ESP32, Arduino…) optimisés pour la faible consommation et la fiabilité 24h/24.

Un système IoT complet, c’est une chaîne de bout en bout : capteurs, microcontrôleurs, protocoles légers (MQTT, CoAP, LoRaWAN) et plateformes cloud pour exploiter ces données.

Sur WaytolearnIoT, tu apprends ce qui se passe « sous le capot » : architectures CPU RISC, gestion mémoire, RTOS et communication embarquée.

Architectures Microprocesseurs

Comprenez les différences entre les architectures RISC et leurs applications dans l'IoT.

Pourquoi c'est important : Les microcontrôleurs IoT (ARM Cortex-M, RISC-V) sont optimisés pour la faible consommation et la prédictibilité plutôt que la puissance brute.

ARM Cortex-M (RISC)

  • ✓ Cycles/Instruction : 1-3 (ultra rapide)
  • ✓ Jeu d'instructions : ~200 instructions simples
  • ✓ Énergie : 0.5 - 2 mW en mode actif
  • ✓ Utilisation : Arduino, ESP32, STM32
  • ✓ Avantage : Prédictibilité temps réel, low-power

RISC-V (Open Source)

  • ✓ Cycles/Instruction : 1-4 (comparable ARM)
  • ✓ Jeu d'instructions : Modulaire, extensible
  • ✓ Énergie : 0.3 - 1.5 mW (ultra-low-power)
  • ✓ Utilisation : Research, custom IoT, edge
  • ✓ Avantage : Open standard, licence gratuite

Systèmes d'exploitation Temps Réel (RTOS)

Un RTOS garantit que les tâches critiques s'exécutent dans des délais prévisibles, ce qui est essentiel pour de nombreux systèmes IoT.

Priorité Déterministe

Les tâches sont exécutées selon leur criticité, pour éviter les retards sur les fonctions vitales.

Gestion des Tâches

Allocation efficace du CPU et basculement rapide entre tâches (context switching).

Synchronisation

Sémaphores et mutex pour éviter les conditions de course et protéger les ressources partagées.

Protocoles de Communication IoT

L'IoT repose sur des protocoles légers comme MQTT et CoAP pour fonctionner sur des réseaux limités en bande passante.

Contexte Sénégal : LoRaWAN est adapté aux zones rurales, avec plusieurs kilomètres de portée et une autonomie de batterie de plusieurs années.

MQTT

Protocole Publisher/Subscriber fonctionnant sur TCP/IP, très utilisé pour la télémétrie et les notifications IoT.

  • • Avantages : Faible overhead, gestion du QoS.
  • • Utilisation : AWS IoT, Azure IoT Hub, brokers open source.

CoAP

Protocole RESTful sur UDP, conçu pour les appareils contraints et les réseaux à forte latence.

  • • Avantages : Modèle Requête/Réponse, paquets compacts.
  • • Utilisation : Domotique, réseaux locaux, 6LoWPAN.
