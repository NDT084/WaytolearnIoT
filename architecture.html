Architecture CPU : Le Cœur de l'IoT

Comprendre l'architecture RISC (ARM & RISC-V) pour optimiser les performances énergétiques.

Objectifs pédagogiques : Différence RISC/CISC, pipeline d'exécution, analyse assembleur pour optimiser les performances, et choix d’architecture selon les contraintes énergétiques.

1. L'ère du RISC : ARM Cortex-M vs. RISC-V

ARM Cortex-M

Architecture ultra répandue dans les microcontrôleurs, optimisée pour le coût, la taille et surtout la faible consommation énergétique, idéale pour capteurs et systèmes sur batterie.

Exemples :

  • • STM32F4 : Cortex-M4 à 168 MHz.
  • • ESP32 : WiFi/Bluetooth intégré, très utilisé en IoT.
  • • Arduino Nano 33 BLE : Cortex-M4 pour l’apprentissage.
  • • ISA : Jeu d'instructions réduit (RISC).
  • • Pipeline : 3 à 5 étages pour la prédictibilité temps réel.
  • • IoT : Mode Deep Sleep et gestion fine des interruptions.

RISC-V (Open Standard)

ISA ouverte et libre, extrêmement modulaire, permettant d’ajouter uniquement les extensions nécessaires pour des applications IoT ciblées et très efficaces.

  • • Modularité : Modules I, M, A, F, D, C selon les besoins.
  • • Avantage IoT : Pas de licence et personnalisation pour l’Edge Computing.
  • • Adoption : De plus en plus présent dans les nouveaux MCUs IoT.

2. L'Importance du Pipeline et du Registre

Le pipeline permet d’exécuter plusieurs étapes d’instructions en parallèle. En IoT, un pipeline court (3–5 étages) réduit les stalles et rend le temps d’exécution des tâches plus prévisible.

Concept clé : Le Pipeline

Comme une chaîne de montage : pendant qu’une instruction est exécutée, la suivante est décodée et la suivante est chargée. Cela augmente le débit même si chaque instruction prend plusieurs cycles.

Exemple : Avec 3 étages, on peut terminer 1 instruction par cycle une fois le pipeline rempli, au lieu de 3 cycles par instruction.

Le saviez-vous ? Jeu d'instructions Thumb

Les cœurs ARM utilisent souvent Thumb (16 bits) en plus de l’ARM 32 bits. Thumb réduit la taille du code, ce qui économise de la Flash et améliore la densité, crucial sur microcontrôleurs.

3. Tutoriel Pratique : Compter les Cycles

Tuto : Analyse du Code en Assembleur (Cortex-M)

Comprendre comment une ligne de C se traduit en instructions est la base de l’optimisation de cycles.

Compilation & analyse :

arm-none-eabi-gcc -S -O2 code.c
// Génère code.s (assembleur)
arm-none-eabi-objdump -d code.o
// Affiche le code machine

Code C :

int a = 5;
int b = 10;
int result = a + b;

Traduction (simplifiée) :

MOV R0, #5    ; 1 cycle
MOV R1, #10   ; 1 cycle
ADD R2, R0, R1; 1 cycle

Performance : environ 3 cycles d’horloge pour l’opération.

Exercice : Estimer le temps d'exécution

Objectif : Calculer le nombre minimal de cycles.

Un microcontrôleur RISC (pipeline 3 étages) exécute :

  • Instruction 1 (Load) : 3 cycles
  • Instruction 2 (Add) : 1 cycle
  • Instruction 3 (Store) : 3 cycles

Question : Temps total si le pipeline est parfait (pas de stalle) ?

Afficher la solution

Réponse : Temps = durée de la première instruction (3 cycles) + 1 cycle pour chacune des 2 autres (chevauchement).
Temps total = 3 + (3 - 1) = 5 cycles.
