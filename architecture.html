Architecture CPU : Le Cœur de l'IoT

Comprendre l'architecture RISC (ARM & RISC-V) pour optimiser les performances énergétiques.

Objectifs pédagogiques : À la fin de ce chapitre, vous serez capable de (1) expliquer la différence entre RISC et CISC, (2) comprendre le fonctionnement d'un pipeline d'exécution, (3) analyser le code assembleur pour optimiser les performances, et (4) choisir l'architecture appropriée selon les contraintes énergétiques de votre projet IoT.

1. L'ère du RISC : ARM Cortex-M vs. RISC-V

ARM Cortex-M

L'architecture la plus répandue dans les microcontrôleurs (MCUs). Son design est optimisé pour les contraintes de coût, de taille et, surtout, de faible consommation énergétique. Essentiel pour les capteurs et les dispositifs fonctionnant sur batterie.

Exemples concrets :

  • • STM32F4 : Cortex-M4 à 168 MHz, utilisé dans les projets IoT industriels
  • • ESP32 : Dual-core Cortex-M, WiFi/Bluetooth intégré, très populaire
  • • Arduino Nano 33 BLE : Cortex-M4, idéal pour l'apprentissage
  • • ISA : Jeu d'instructions réduit et fixe (RISC).
  • • Pipeline : Souvent 3 à 5 étages (plus court que les PC) pour une prédictibilité en temps réel.
  • • Caractéristiques IoT : Support natif du mode veille profonde (`Deep Sleep`) et gestion efficace des interruptions.

RISC-V (Open Standard)

Un jeu d'instructions (ISA) ouvert et libre. Il offre une modularité unique, permettant aux concepteurs d'ajouter uniquement les extensions nécessaires. Cela le rend extrêmement adaptable à des applications IoT très spécifiques, en maximisant l'efficacité énergétique.

  • • Modularité : L'ISA est basé sur des modules de base (I, M, A, F, D, C).
  • • Avantage IoT : Absence de licence et personnalisation pour créer des cœurs micro-spécifiques pour les tâches Edge Computing.
  • • Adoption : De plus en plus présent dans les nouvelles générations de MCUs IoT.

2. L'Importance du Pipeline et du Registre

Le Pipeline d'un microcontrôleur est essentiel : il permet d'exécuter plusieurs étapes d'instructions simultanément. Dans l'IoT, un pipeline court (3 à 5 étages) est préféré, car il minimise le risque de stalles (blocages) et rend le temps d'exécution des tâches prévisible, une nécessité pour le temps réel.

Concept clé : Le Pipeline

Imaginez une chaîne de montage : pendant qu'une instruction est en phase d'exécution, la suivante est déjà en phase de décodage, et celle d'après est en phase de récupération. Cela permet d'exécuter plusieurs instructions en parallèle, augmentant le débit (throughput) même si chaque instruction individuelle prend plusieurs cycles.

Exemple : Un pipeline à 3 étages peut traiter 3 instructions simultanément. Si chaque instruction prend 3 cycles, le pipeline permet de terminer une instruction par cycle après le remplissage initial, au lieu d'attendre 3 cycles par instruction.

Le Saviez-vous ? Le jeu d'instructions Thumb

Les cœurs ARM utilisent souvent le jeu d'instructions Thumb (16 bits) en plus de l'ARM standard (32 bits). Thumb réduit la taille du code (économie de mémoire Flash) et améliore la densité du code, deux facteurs critiques pour les petits systèmes embarqués.

3. Tutoriel Pratique : Compter les Cycles

Tuto : Analyse du Code en Assembleur (Cortex-M)

Comprendre comment une ligne de C se traduit en instructions est la clé de l'optimisation. Nous allons analyser le code pour minimiser les cycles d'horloge.

Comment compiler et analyser :

arm-none-eabi-gcc -S -O2 code.c
// Génère code.s (assembleur)
arm-none-eabi-objdump -d code.o
// Affiche le code machine

Code C :

int a = 5;
int b = 10;
int result = a + b;

Traduction (simplifiée) :

MOV R0, #5    ; 1 cycle
MOV R1, #10   ; 1 cycle
ADD R2, R0, R1; 1 cycle

Performance : L'opération prend seulement 3 cycles d'horloge.

Exercice d'application : Optimisation ISA

Objectif : Calculer le nombre minimal de cycles.

Un microcontrôleur RISC (3-étages pipeline) exécute la séquence suivante :

  • Instruction 1 (Load) : 3 cycles
  • Instruction 2 (Add) : 1 cycle
  • Instruction 3 (Store) : 3 cycles

Question : Quel est le temps total d'exécution si le pipeline est parfait (pas de stalle) ?

Afficher la solution

Réponse : Dans un pipeline parfait, les étapes se chevauchent. Le temps total est égal au temps d'exécution de la première instruction (3 cycles) plus 1 cycle pour chacune des instructions restantes (2 instructions).
Temps = 3 + (3 - 1) = 5 cycles.
